View source on that page in your browser. Look at the option tags that are generated.

HTML Code:
<option value="myValue">shown value</option>
Whatever is assigned to the option value is returned by the browser as a string literal. Likely, that string literal is something like "Unit@15!2%", the result of your Unit object's toString() method.

What you want assigned to the value attribute is some sort of ID that you can then use to reference the appropriate Unit object. If it's a number, it can be a primitive int, Integer, or String representation - so long as you map it to a command object property that is of type Integer or String. In other words, command.equipment.unit needs to be Integer or String. You should probably rename it to command.equipment.unitID just to be clear.

This is roughly what your jsp code would look like:

HTML Code:
<spring:bind path="commandName.equipment.unitID">
  <select name="${status.expression}">
    <c:forEach var="entry" items="${unitList}">
      <option value="${entry.ID}"> ${entry.name}</option>
    </c:forEach>
  </select>
</spring:bind>

If you want to bind Objects and not primitives you will have to use a PropertyEditor, which converts the selected String into a Object.

Also you seem to want to use the collection of numbers as reference data and not as a part of the command object.

To put it simply the commandObject (or form backing object) contains only the data entered in the form!

Quote:
Does anybody else agree that good docs for Spring MVC are impossible to find?
No. Check the reference guide and the samples shipped with Spring they provide a lot of information. For more in depth/technical information check the javadocs of the Spring MVC framework.

For reference data to be included implement the referenceData method

Code:
protected Map referenceData(HttpServletRequest request) throws Exception {
  Map data = new HashMap();
  List numberList = new ArrayList(); //Should be replaced by getting the list of numbers
  data.put("numbers", numberList);
  return data;
}
The above part makes the list of numbers available under the name numbers and is thus accessible with ${numbers}.

In your commandObject create a property number with the correct get/set pair.

Code:
public class NumberObject {
  private String number
  public void setNumber(String number) {
    this.number=number;
  }
  public String getNumber() {
    return this.number;
  }
}
Next change your code in the jsp to use the correct collection and property in the command object. Here is an example with the new form library

Code:
<form:select path="number">
   <form:option value="" label="--Please choose--">
   <form:options items="${numbers}"/>   
</form:select>
<form:errors path="number"/>
