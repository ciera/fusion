The value expression is evaluated against the RequestContext, so you'll have to use the flowScope prefix:

Code:
<set attribute="form1.property1" value="${flowScope.form2.property1}" />

The problem is that you're trying to set the "property1" of bean "form1" in flow scope, but "form1" is not in flow scope (yet). That's why the exception says:

Quote:
target is null for setProperty(null, "property1", value1)
The target object is "form1".

To set nested properties, make sure the base target bean is available (not null).
