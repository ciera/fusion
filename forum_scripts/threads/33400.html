You might want to enclose you ${roleElements.roleName} inside '' which makes it a key if I'm not mistaken.

Quote:
Originally Posted by Hippo
One other thing that I tried was forEach.
Looking at your code I assume you are using a forEarch to iterate over your collection of roles. But apparently your not.

Also you want to display the rolename (your c:out) I assume instead of the value/description in your map. I guess the roles which are selected are already in the map? The value of the checkbox should be the Id of the role, not just simply true of false. Only the selected checkboxes will be sent back to the controller. In the controller you need some way to determine which roles are checked. If the value was just 'true' and you selected 4 items you would get 4 times true returned without knowing which roles where selected.

Also you would need a PropertyEditor to convert the Id/rolename or whatever back ro a role.

The snippet in the jsp should look something like this. I just made up the description and id property of the Role object.

Code:
<c:forEach var="role" items="roleElements"
  <spring:bind path="roles['${role.roleName}']" >
    <td><c:out value="${role.description}" /></td>
    <td><input type="checkbox" name="<c:out value="${status.expression}"/>" value="${role.id}" <c:if test="${status.value}">checked</c:if> />
    </td>
  </spring:bind>
</c:forEach>

Well, with a smile on my face I anounce that the problem is solved.

This part does binding. No change here.
Code:
<spring:bind path="rolesMap['${roleElements.roleName}']" >
  <td><c:out value="${roleElements.roleName}" /></td>
  <td>
        <input type="hidden" name="_<c:out value="${status.expression}" />" />
	<input type="checkbox" name="${status.expression}" value="true"
	<c:if test="${status.value == roleElements.roleId}" >checked</c:if> />
  </td>
</spring:bind>
