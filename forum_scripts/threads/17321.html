Where to do a removeAll
I have a master list of users and then several lists of users linked to my object. I retrieve the master list in the referenceData of my SimpleFormController class and the lists of already-linked users in the formBackingObject function. I would like to remove all of the already-linked users from the master-user list. What is the best way to do this? I have considered a scriptlet but I know that's a 'no-no.'

referenceData is called after formBackingObject. Override:
Code:
	protected Map referenceData(HttpServletRequest request, Object command, Errors errors) throws Exception {
}
So you could put two sets into the map, one for all users, and one for unselected users.

Ooohhh, instance variable. Big no no Your controller *isn't* explictly thread safe, so you need to make it be. Having statful instance variables is the quickest way to ensure it isn't

If you really really want to have an instance variable, then synchonise access to it. Basically you can (of course) have them, but be aware of their consequences (i.e. thread safety).
