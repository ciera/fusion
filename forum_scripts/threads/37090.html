It looks like your 'spring:bind' tags are not necessary, since you are using the 'form' tags. For the form tag, the 'path' attribute should be relative to the command (as that is already specified on the top level 'form:form' tag) - for example use path='memberType' instead of path='user.memberType'.

Hope that helps.

The "goto" controller is not necessary. The default behavior of a Spring form controller is to automatically create the form-backing object (of type User.class in your case) for an incoming request. If you perform a GET request on '/register.html', that command object should be created and the form view displayed.

On line 26 of your validator, you should replace "user.getFirstName()" with the name of the property (just 'firstName'). For example:

From:
Code:
ValidationUtils.rejectIfEmptyOrWhitespace(errors, user.getFirstName(), "firstName.required", "First Name is required");
to:
Code:
ValidationUtils.rejectIfEmptyOrWhitespace(errors, "firstName", "firstName.required", "First Name is required");
