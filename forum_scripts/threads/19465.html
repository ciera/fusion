Can a subflow return an unknown object type?
Hey All,

I have a subflow which returns an output credit card. This is from the parent flow:

Code:
	<subflow-state id="paymentMethod" flow="paymentMethodFlow">
		<attribute-mapper>
		    <output value="${flowScope.paymentMethod.creditCard}" as="creditCard"/>
	   </attribute-mapper>
   		<transition on="success" to="confirmGiftCertificate"/>
		<transition on="previous" to="selectGiftCertificate"/>
		<transition on="error" to="error"/>
	</subflow-state>
If the parent doesn't set a dummy "creditCard" attribute into the flow context before this subflow then I get an error when it returns. Is there any way to tell SWF to create the attribute "creditCard" and set it with the value if it doesn't exist?

This "variable resolution" capability is not possible at present. It sounds like a good idea though, similiar to what JSF does.

The tricky thing is how does SWF know what class a paymentMethod is? And then what class a creditCard is? That metadata has to be provided somewhere, and then interpreted by the expression evaluation facility. This is not yet supported.

Could you open a JIRA issue?

Right, logically SWF does this during the parent flow resume operation:

Code:
getFlowScope().getAttribute("paymentMethod").setCreditCard(creditCard);
But if paymentMethod doesnt exist as an attribute, how the heck does it know what to do? That's what I'm saying: without a paymentMethod bean in flow scope already, SWF doesn't have enough information to go on: there is currently not support to automatically create a new PaymentMethod bean and then set the credit card if a PaymentMethod doesn't already exist.

As long as PaymentMethod already exists in parent flow scope it should work fine with PR5 or >.

Make sense?
