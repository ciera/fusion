Using abstract classes or interfaces as the types of properties on your form object or anywhere else in the object graph should not make a difference from the point of view of the data binder.

I tried the following:

Code:
package foobar;

import java.io.Serializable;

public class Foo implements Serializable {

    private String prop;

    public String getProp() {
        return prop;
    }

    public void setProp(String prop) {
        this.prop = prop;
    }
}
Code:
package foobar;

import java.io.Serializable;

import org.springframework.beans.BeanWrapperImpl;

public class Bar {

    private Serializable foo;

    public Serializable getFoo() {
        return foo;
    }

    public void setFoo(Serializable foo) {
        this.foo = foo;
    }

    public static void main(String[] args) {
        Bar bar = new Bar();
        Foo foo = new Foo();
        foo.setProp("FooBar");
        bar.setFoo(foo);

        System.out.println(new BeanWrapperImpl(bar).getPropertyValue("foo.prop"));
    }

}
And this prints "FooBar" as expected.

What error are you getting exactly?
