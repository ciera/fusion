Object is null on command in view state despite being set by action state
Hi

I have a flow into which a request parameter is passed. Based on this parameter, I call a service via an action state to set something on my command.

I then transition on to a view state.

In my view state the object reference that I just set on the command action state is now null.

It is almost as if the command is at the request level not flow level.

Here is a snippet of my flow:

Code:
<?xml version="1.0" encoding="UTF-8"?>

<flow xmlns="http://www.springframework.org/schema/webflow"
	xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xsi:schemaLocation="http://www.springframework.org/schema/webflow
                          http://www.springframework.org/schema/webflow/spring-webflow-1.0.xsd">

	<input-mapper>
		<mapping source="code" target="flowScope.code" />
	</input-mapper>


	<start-state idref="setSelectedTableOnForm" />



	<action-state id="setSelectedTableOnForm">
		<action bean="editTableDataFormAction"
			method="setSelectedTableOnForm" />
		<transition on="success" to="editTableData" />
	</action-state>

	<view-state id="editTableData" view="editTableData">

		<render-actions>
			<action bean="editTableDataFormAction" method="setupForm" />
		</render-actions>
		<transition on="saveTable" to="save" />
	</view-state>
Here is the code that sets the object on the command

Code:
/**
	 * sets the selected table on the form

If I try and map to and from code and description fields in a JSP using spring:bind or the new Spring 2 form:input tags it does not carry over the values from a to b.

It seems that the mapping code within Spring does not work properly for properties on the superclass.

This must be some sort of configuration or classpath problem on my part because this would be a pretty fundamental flaw otherwise, any help greatly appreciated.

I did a quick sanity test and the situation you describe seems to work correctly:

Base entity class:
Code:
package other;

import java.io.Serializable;

public abstract class BaseEntity implements Serializable {
	
	private String code;
	private String description;
	
	public String getCode() {
		return code;
	}
	
	public void setCode(String code) {
		this.code = code;
	}
	
	public String getDescription() {
		return description;
	}
	
	public void setDescription(String description) {
		this.description = description;
	}

}
Concrete entity:
Code:
package other;

public class Person extends BaseEntity {
	
	private String name;
	
	public String getName() {
		return name;
	}
	
	public void setName(String name) {
		this.name = name;
	}

}
Test code:
Code:
package other;

import java.util.HashMap;
import java.util.Map;

import org.springframework.webflow.action.FormAction;
import org.springframework.webflow.core.collection.LocalParameterMap;
import org.springframework.webflow.execution.RequestContext;
import org.springframework.webflow.execution.ScopeType;
import org.springframework.webflow.test.MockRequestContext;

public class Test {
	
	public static void main(String[] args) throws Exception {
		
		Map requestParams = new HashMap();
		requestParams.put("code", "ABC");
		requestParams.put("description", "Testing 123");
		requestParams.put("name", "Foo");

		FormAction formAction = new FormAction();
		formAction.setFormObjectClass(Person.class);
		formAction.setFormObjectName("person");
		formAction.setFormObjectScope(ScopeType.FLOW);

		RequestContext context = new MockRequestContext(new LocalParameterMap(requestParams));
		formAction.bind(context);
		
		Person person = (Person)context.getFlowScope().get("person");
		
		System.out.println(person.getCode());
		System.out.println(person.getDescription());
		System.out.println(person.getName());
	}

}
And the resulting output is as expected:
Code:
ABC
Testing 123
Foo
