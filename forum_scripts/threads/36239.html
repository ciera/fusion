Well I would say that definitly is the problem. Because at render time (before executing of the javascript) all the possible values are checked against the value in the command object. If you are filling the values after rendering then no selection is possible anymore. At least nog by Spring. You would have to modify your javascript for that.

But why not just use Spring to do that for you?

Put the values as a list in the mav and you should be able to render it without the use of javascript...
