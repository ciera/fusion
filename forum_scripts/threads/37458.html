Just override the initBind method and the suppressValidation method (assuming you are using Spring 2.0.4 or greater)

Code:
protected void initBinder(HttpServletRequest request, ServletRequestDataBinder binder) throws Exception

    binder.setRequiredFields(new String[] {"name", "firstName", "gender", "firstName"});
By default normal validation still occurs, so if you have registered some other validators they will still be called. If you want to prevent this override the suppressValidation method also.

Code:
protected boolean suppressValidation(HttpServletRequest request, Object command, BindException errors) {
    if (errors.hasErrors) {
        return true;
    }
    return super.suppressValidation(request, command, errors);
}
That should do the trick.

Quote:
and if this works, how can i find out in the jsp-file, which field was blank???
Spring will generate messagecodes for the missing fields. For example required.name or required.gender. If you use spring 2.0 you can simply use the form:errors tag. It will resolve the message for you. Check the DefaultMessageCodesResolver for an explaination of which codes are generated.

Quote:
He thinks all these pages are already need in the first page. But they are need in page three... How can i tell this funktion that i only should check this on page three??
Check the API of the AbstractWizardFormController it has a very convenient getCurrentPage method.

Code:
protected void initBinder(HttpServletRequest request, ServletRequestDataBinder binder) throws Exception
	if (getCurrentPage(request) == 3) {	
    		binder.setRequiredFields(new String[] {"name", "firstName", "gender", "firstName"});
	}
}

    private boolean PasswordsAreEqual(String password, String repeatedPW)

    	if (password == null)
