transition on-exception not working
Hi I'm using the following view-state in the flow:

Code:
<view-state id="enterQuantity" view="enterQuantity2">
        <transition on="back" to="selectFoodItems"/>
        <transition on="next" to="displayResults">
            <action bean="groupCategoryAction" method="bind"/>
        </transition>
        <transition on-exception="java.lang.IllegalArgumentException" to="enterQuantity"/>
    </view-state>
during the binding process, the backing object will throw IllegalArgumentException if quantities entered is <0. So I figure I'll just use on-exception to pick up this exception and bounce the flow back to the current state. Thus my intention is to cycle the state until all values>0 i.e. no more java.lang.IllegalArgumentException.

However during execution I get the following:
Code:
root cause

java.lang.IllegalArgumentException: contents cannot be less than zero!

Anyone can lend me a hand on this? I do have a feeling that maybe this on-exception can only catch exceptions in the state but not those exceptions generated from the transition clause. But how do you make WebFlow react to exceptions generated from actions done inside the <transition> clause?

Thanks for the advice. It worked. I override bind() and made this work:

Code:
@Override
    public Event bind(RequestContext context) throws Exception {
        Event event = null;
        try {
            event=super.bind(context);
        } catch (IllegalArgumentException ex) {
            return new Event(this,"error");
        }
        return event;
    }
