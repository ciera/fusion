Complete Struts-SWF integration [1.0RC1]
I think i got it working with 1.0RC1 and a wizard style navigation pages [back/next buttons]
the key is the flow xml file, that you would need to define to construct your flow..i am not sure about the abstract flow builder class but it think i like the XML a lot better than a class...
Code:
<flow start-state="setupForm">

	<!-- this is important state since it binds the object to the action..
               The action state is persisted for the entire flow, so this method                 
                will reset the bean only once
         -->
	<action-state id="setupForm">
		<action bean="facInfoAction"/>
		<transition on="success" to="enterFacXXXXInfo"/>
	</action-state>
	
	<!--
		Display the input form view.
	-->
	<view-state id="enterFacXXXXInfo" view="enterFacXXXXInfo">
		<transition on="submit" to="facInfobindAndValidate"/>
	</view-state>
	
	<!--
		Bind and validate input data from the form. 
	-->
	<action-state id="facInfobindAndValidate">
		<action bean="facInfoAction" method="bindAndValidate"/>
		<transition on="success" to="enterFacilityAdmin"/>
		<transition on="error" to="enterFacilityInfo"/>
	</action-state>
	
        <!--
		I have to create a setup method for each navigation page. 
                The reason will become apparent later on when you will see the  
                 back button functionailty
	-->
        <action-state id="setupFormFacAdmin">
		<action bean="facAdminAction" method="setupForm" />
		<transition on="success" to="enterFacilityAdmin"/>
	</action-state>
        
	<view-state id="enterFacilityAdmin" view="enterFacilityAdmin">
		<transition on="submit" to="facAdminbindAndValidate"/>
         <!-- the setupForm is called to initialize the first page in the flow -->
                <transition on="back" to="setupForm" />
	</view-state>
        
        <action-state id="facAdminbindAndValidate">
		<action bean="facAdminAction" method="bindAndValidate"/>
		<transition on="success" to="enterOpCert"/>
		<transition on="error" to="enterFacilityAdmin"/>
	</action-state>
        
        <view-state id="enterOpCert" view="enterOpCert">
		<transition on="submit" to="opCertBindAndValidate"/>
         <!-- setupFormFacAdmin is called because you just want to travel 
                back to the previous page
          -->
               <transition on="back" to="setupFormFacAdmin" />
	</view-state>
	
<!-- do it for the rest -->
now to the context file...the key here is persisting the action for the entire flow

Code:
<!-- beging with giving the actual XML -->
<bean id="flowLocator" class="org.springframework.webflow.registry.XmlFlowRegistryFactoryBean">
		<property name="flowLocations">
			<list>
			 	<value>/WEB-INF/form####.xml</value>			 	
			</list>
		</property>
	</bean>
       <!-- you want to define it for entire scope.. 
             since most of my code is in validator and in the final action, 
             there is no need for me to create individual actions that extend FormAction. 
             So i am configuring everything in the XML
        -->
	<bean id="flowScope" class="org.springframework.beans.factory.config.FieldRetrievingFactoryBean">
            <property name="staticField"><value>org.springframework.webflow.ScopeType.FLOW</value></property>
        </bean>
        
	<bean id="facInfoAction" class="org.springframework.webflow.action.FormAction">
            <property name="formObjectName"><value>xxxxInfo</value></property>
            <property name="formObjectScope" ref="flowScope"/>
            <property name="formObjectClass">
                <bean class="java.lang.Class" factory-method="forName">
                        <constructor-arg><value>test.wizard.beans.TestBean</value></constructor-arg>
                </bean> 
            </property>
            <property name="validator"><bean class="test.wizard.validators.XXXXXValidator" /></property>
        </bean>

in the previous post, you already know what you need to put in each JSP page...
make sure the action remains the same in each individual JSP page..
i.e.
Code:
<html:form action="/wizardAction">
to start a new wizard or flow, pass the parameter
Code:
<html:link action="/wizardAction?_flowId=form2272
this way you could pass any type of flow by simply replacing the XML file....
doing this will allow your users to go back and forth using the back-next button usually used in car insurance quotes, air line ticket booking etc...and you didnt even write any code..
if you think validators are too much code as well, use valang...

using web-flow definitely you can reuse components... great job keith and the team
hope this helps

Thanks, Spring Guy.

BTW -- just a few pointers from your code:

No need to use a FieldRetrievingFactoryBean to specify flow scope. Simply do this:
Code:
<property name="formObjectScope" value="FLOW"/>
... Spring will convert that to the ScopeType.FLOW constant automatically for you :-)

Also, no need to construct a class using a factory method--simply specify the class name value and rely on spring's built in property editor machinery to perform the conversion:

Code:
<property name="formObjectClass" value="example.MyClass"/>
