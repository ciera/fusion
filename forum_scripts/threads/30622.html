WebSphere / Spring 2.0 / Web Flow 2.0 nightly build
Hello,

I've encountered a problem when migrating my application to Spring 2.0 and the Web Flow Oct-23 nightly build. My application is deployed to WebSphere 6.0.2.5, within the Rational Applicaton Developer IDE.

My applcatiion worked fine with Spring 2.0-RC2 and Web Flow 1.0-RC3. After upgrading to the final 1.0 release of Spring, which requires Web Flow 1.0-RC4 or newer, I'm getting java.lang.NoSuchMethodError Exceptions when my XML schema configuration is parsed at application startup.

I first thought that only <jee:jndi-lookup> was producing the error, because I previously used a generic <bean id="dataSource"> definition. However, reverting the <jee:jndi-lookup> back to the generic definition causes yet another XML-schema definition to fail; this time it's with <flow:executor>.

Code:
	<jee:jndi-lookup id="dataSource" jndi-name="jdbc/abc"/>
Code:
	<!-- Launches new flow executions and resumes existing executions. -->
	<flow:executor id="flowExecutor" registry-ref="flowRegistry" repository-type="singlekey">
		<flow:execution-attributes><flow:alwaysRedirectOnPause value="false"/></flow:execution-attributes>
		<flow:execution-listeners><flow:listener ref="flowListener"/></flow:execution-listeners>
	</flow:executor>

	<!-- Creates the registry of flow definitions for this application -->
	<flow:registry id="flowRegistry">
		<flow:location path="/WEB-INF/**/flows/*-flow.xml" />
	</flow:registry>
With Spring 2.0-RC2 and Web Flow 1.0-RC3 I used XML-schema tags to configure transactional advice, both <tx:advice> and <aop:config>. I'm confused as to why I can't use <jee:jndi-lookup> or <flow:executor> with the latest versions of Spring 2.0 and Web Flow 1.0.

Other support threads have commented about issues of using XML schema with JDK 1.4. I added xercesImpl-2.8.1.jar and xml-apis-1.3.03.jar to my WEB-INF and configured them to load before my "JRE System Library" jars defined in the WebSphere v6 JRE.

I don't imagine I'm the only developer deploying to WebSphere 6.0.2.5, or a more recent release. Have any other WebSphere developers found an issue with the final release of Spring 2.0 or Web Flow-RC4?

Thanks in advance.
