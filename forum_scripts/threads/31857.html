Initializing form object before setup
Hi all,
I'm trying to make the method="setupForm" of a FormAction bean integrate data already coming from a previous state: how can I do that?

My jsp page includes an HTML form with payment-related fields: some of them are hidden and come from a configuration file (like the shopId) while others are to be filled-in by the customer through the form. I have a scope-level object carrying on the configuration-fields but I don't know how to have these mixed to the data set up by the FormAction object. If I enter the configuration-fields into the HTML form, they are not going to be linked in any way to the new data inserted by the customer. In fact, upon submit, the _flowId appears to be lost and this causes a FlowExecutorArgumentExtractionException to be thrown.

This is the definition of my view-state:
<view-state id="payment" view="paymentView">
<bean-action bean="bookingService" method="getPaymentRequest">
<method-result name="paymentRequest" scope="flow" />
</bean-action>
<action bean="paymentFormAction" method="setupForm" />
<transition on="cancel" to="leave" />
</view-state>

I would like to have the paymentRequest object used in the setupForm.

In you jsp you need a mechanism to include the flowExecutionKey. I tend to mostly use hidden fields.

Code:
<input type="hidden" name="_flowExecutionKey" value="${flowExecutionKey}"/>
In your PaymentFormClass you can override the createFormObject method, by default the formObject is placed in flowScope, all other actions to your bean in the same flow will use this instance of the formobject. Unless you call resetForm which will create a new one.

Code:
protected Object getFormObject(RequestContext context) throws Exception{ 
   PaymentRequest formObject = bookingService.getPaymentRequest();
   return formObject;
}
Also yuo don't have specified what needs to happen on a submit, only on what needs to happen on leave. When you press submit you need to tell your flow that is has to bind the values from the form to your formObject. You can do this by calling the bind (or bindAndValidate) method.

Code:
<view-state id="payment" view="paymentView">
    <render-actions>
        <action bean="paymentFormAction" method="setupForm" />
    </render-actions>
    
    <transition on="save" to="nextStep">
        <action bean="paymentFormAction" method="bind"/>
    </transition>
    <transition on="cancel" to="leave" />
</view-state>
Take a look at the examples shipped with Spring Web Flow the give a nice overview on how to handle things.

Quote:
What I think I'm still missing is the way to link the pre-existing paymentRequest object to the formAction.
This should be straightforward, just make sure you put the pre-existing form object in a place where the FormAction will look for it. Given your FormAction config, if you have a seperate action state that runs the following action code *before* your view state which uses FormAction to setup the form object:

Code:
...
PaymentRequest formObject = bookingService.getPaymentRequest();
new FormObjectAccessor(context).putFormObject(formObject, "paymentRequest", ScopeType.FLOW);
...
Your FormAction will pickup this form object and bind to it, basically completing your form object with data submitted by the user.

I tested this by modifying the phonebook sample that comes with SWF and it works as expected. There must be some silly mistake somewhere that we're missing.
One thing to look out for is the fact that you've got the call to "getPaymentRequest" in an entry action. This means that if the bindAndValidate would fail the "payment" view-state will re-enter and as a result the "getPaymentRequest" will be executed again, basically resetting the form object. So you probably want to move the "getPaymentRequest" bean action to a seperate action state.

        <input type="submit" value="Cancel" name="_eventId_cancel">				
      </td>
    </tr>
  </table>
</spring:nestedPath>
