[Struts+Spring+SWF] Problem in managing continuations
Hi,
We are using Struts and Spring in our application development.

While evaluating SWF for implementing wizard based screens, I came across problem in managing continuation in the flow (i.e. the click of 'refresh', 'back', 'forward' button in the browser window make the application to crash ). This also happens in the 'birthDate' example distributed as SWF samples. I observed that 'birthDate' example doesn't makes use of 'flowExecutor' which is responsible for creating and starting new flow executions as well as signaling events for processing by existing, paused executions.

Therefore, I introduced the following extra piece of configuration in webflow-context.xml of birthDate example:

Code:
<bean id="flowExecutor" class="org.springframework.webflow.executor.FlowExecutorImpl">
	<constructor-arg ref="repositoryFactory"/>
	<property name="redirectOnPause" value="FLOW_EXECUTION"/>
</bean>

<bean id="repositoryFactory" class="org.springframework.webflow.execution.repository.continuation.ContinuationFlowExecutionRepositoryFactory">
	<constructor-arg ref="flowLocator"/>
</bean>
Rest of the file is still the same as distributed with sample application. Upon doing this alteration to the configuration file and running the application, I get the following exception:

What version are you using? You should upgrade to 1.0 RC3 first off.

"redirectOnPause"=true will cause problems if your form object is not in flow scope in 1.0 RC3. Try setting to false with 1.0 RC3 if you still experience this problem.
