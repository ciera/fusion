ResourceBundleMessageSource access within an Action
I'm currently using a MessageSource property file with WebFlow for my form's error messages, however, I'd like to extend this property file to also include messages that are not directly related to form entry. I do not have multiple languages to support (yet), but I'd like to have the capability present. At the moment, I have a single "MessageResources.properties" file and it works fine for my "Error.rejectValue()" calls, however, I can't seem to get my hands on an object that I can use within my main action beans for messages out to log files or the like.

I tried implementing MessageSourceAware in my AbstractAction, however, when I call getMessage on that object, and supply it a default Locale (since I'm only using the default properties file, any Locale should still resolve the "basename" property file) it generates an exception:

Code:
org.springframework.context.NoSuchMessageException: No message found under code 'testmessage' for locale 'en_US'
My ResourceBundleMessageSource is configured like:
Code:
<bean id="messageSource" class="org.springframework.context.support.ResourceBundleMessageSource">
    <property name="basename">
        <value>MessageResources</value>
    </property>
</bean>
And my MessageResources.properties file has (among many other properties used by my forms):
Code:
testmessage=Lorem ipsum dolor sit amet
The code I've been working with is:
Code:
msgSrc.getMessage("testmessage", null, Locale.getDefault());
Any ideas as to why this doesn't seem to be resolving the "basename" property file when used in this context?

My guess would be that you're defining your MessageSourceAware action in a Spring beans file included from the web flow XML while you're defining the ResourceBundleMessageSource in the main DispatcherServlet application context.

Try to move the action bean definition to the DispatcherServlet application context.

Not necessarily the 'file', but the application context. As you know a single application context can be made up of bean definitions in multiple XML files that are merged together.

Quote:
Beans in both flows, defined within their respective independent context configuration files, can utilize the MessageSource in the servlet context for forms-based error messages. Those same beans, however, cannot access the same MessageSource for non-error message operations.
So they can access the MessageSource but don't see all messages? Make sure the MessageSource can find all .properties files in the classpath.

Intriguing... of course I write out a lengthy explanation of the issue only to realize it has resolved itself. I do have multiple context configuration files, one for "global" beans, and two separate ones for individual flows, to maintain organization. When I originally had the "messageSource" bean in the "global" context file, the property file would only be resolved with regard to form field errors (via the Error.rejectValue() method). Any attempts to call "getMessage" from a MessageSource injected via MessageSourceAware wouldn't resolve the same properties file. After a restart of Tomcat, it seems to be resolving properly now... which is strange, but I'm not going to argue.
