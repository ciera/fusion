So it seems that I've found the source of this erratic behaviour. I had a "Submit" button on the page, which shared some JavaScript with text links - all of which needed to have the JavaScript perform the page submit (and ultimately the redirect). Unfortunately under IE (and not FireFox) - the submit action was sometimes (about a third of the time) being performed twice, which was causing the flow exception.

By simply changing this to a "Button" input - the problem is solved, since the JavaScript performs the submit anyway. So everything seems happy for now - thanks Erwin!
