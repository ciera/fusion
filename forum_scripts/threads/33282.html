Here some code I have which lauches a flow from a non-spring managed page inside a datatable.

Code:
<h:column>
    <f:facet name="header">
	<t:commandSortHeader columnName="caption" arrow="true" immediate="false">
	    <h:outputText value="Name" />
	</t:commandSortHeader>
    </f:facet>					            
    <h:commandLink value="#{record.caption}" action="flowId:user-find-flow">
	<f:param name="id" value="#{record.user.uuid}" />					            	
    </h:commandLink>
</h:column>
I would imagine it would be nearly the same for your situation, exception you dont need the flowId bit like I have. You want it declared in a hidden field in side the form at the bottom of the page usually.
