As stated before your InternalResourceViewResolver has to be the last in the line. As stated in the javadocs of the InternalResourceViewResolver it ALWAYS returns a view if it exists or not. So switch the order. First your XmlViewResolver.

Quote:
Originally Posted by javadocs
Note: When chaining ViewResolvers, an InternalResourceViewResolver always needs to be last, as it will attempt to resolve any view name, no matter whether the underlying resource actually exists.
Code:
<bean id="pdfViewResolver" class="org.springframework.web.servlet.view.XmlViewResolver">
	<property name="order" value="1"/>
</bean>

<bean id="viewResolver" class="org.springframework.web.servlet.view.InternalResourceViewResolver">
	<property name="viewClass" value="org.springframework.web.servlet.view.JstlView"/>
	<property name="prefix" value="/WEB-INF/jsp/"/>
	<property name="suffix" value=".jsp"/>
	<property name="order" value="2"/>
</bean>
This is something you need if you ever want it to work. The other issue that arises is a bit hard to uncover without those parts of the configuration. Can we see your complete configuration, regarding controllers/views/dispatcherservlet.

Your problem with your resolver is the following controller

Code:
<bean id="index" class="it.mycompany.myapp.controller.IndexController">
	<property name="successView">
		<value>index</value>
	</property>
</bean>
It has to do with the internal working of the ResourceBundleViewResolver and XmlViewResolver. They use the current application context to lookup beans with a certain name, if they don't find a bean with the given name they use the property/xm file to construct one. So when your application is starting/receiving it's first request (when ResourceBundleViewResolver is the first resolver) it looks for a bean with the name index (as that is the configured succesview of your controller) it finds the bean BUT the bean is a Controller and not a view. Hence your error.

Tweaking your configuration would solve it.

Rename your IndexController from index to indexController, change your mapping accordingly and it should work.

Code:
<bean id="indexController" class="it.mycompany.myapp.controller.IndexController">
	<property name="successView" value="index"/>
</bean
Mappings

Code:
<bean id="urlMappingLogin" class="org.springframework.web.servlet.handler.SimpleUrlHandlerMapping">
    [.. removed interceptors ..]
	<property name="mappings">
		<props>
			<prop key="/index.htm">indexController</prop>
			 [.. removed mappings ..]
		</props>
	</property>
</bean>
