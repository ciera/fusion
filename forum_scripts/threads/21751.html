Dynamic Page feed by QueryStrings
Hi everybody,

I need to create a dynamic page (imageTemplate.html) in my spring application, which will include images from different URLs.

In a footer and a header of this specific page the user will have additional some links to the same page to add three different properties via query strings to modify the sources of the images.

Example:

http://server:8080/springapp/imageTemplate.html - shows the template
http://server:8080/springapp/imageTe....html?host=bla - will switch the image source to host "bla"
http://server:8080/springapp/imageTe...?timeframe=12h - will switch the links to the images to show only images for the last 12h

So spring has to remember for example which host was selected and show images for that host only. If a user now selects the link for 12h timeframe, the application should show the links for host "bla" and "12h".

Which is the best way to solve that in spring? I thought about having a bean which contains all properties and use this bean as a command object. But how to feed that command object via Get Query Strings and through several calls?

Thanks in advance!,

Updating a formBackingObject (command) from GET parameters is not an issue; there is no difference between parameters sent via GET or POST (exception length restrictions).

Getting Spring to update the backing object on GETs instead of POSTs is done via overriding AbstractFormController.isFormSubmission to always return true.

The problem is that you are not passing in the parameters every time, so your backing object will need to be stored in the session, otherwise you will loose them and all of this is based on the assumption that imageTemplate is sourced by a SimpleFormController.

Hope this helps

Not sure what you mean
Quote:
will be feed by the properties which are stored in the session


Why not just update the formBackingObject in the session, but do the binding everytime (or at least when there are query string parameters)

As I said originally; it is not very elegant, but ....

Remember, updating an *existing* formBackingObject will only override properties for which request parameters exist. Assume:
Code:
  class PropertyPreferences {
    private String host;
    private String frame;

    // accessors hidden
  }
If this is stored in the session, then:
- first request comes in with "?host=xyz", this is bound and yourBackingObject.setHost("xyz") is called.
- second request comes in with "?frame=abc", this is bound and yourBackingObject.setFrame("abc") is called *but the host is not changed*.

Does that make sense?

No, that wasn't quite what I had in mind

My idea was to put the form backing object on the session, and always do the binding. You don't need to do any of your own binding.

The *only* thing in the session will be your RrdBean.

The other thing; whenever you display a form (JSP) which contains spring:bind tags, the ModelAndView *must* contain the Errors object, which you can get by overriding the onSubmit which passes in the (I think) four parameters, and then adding errors.getModel() to your ModelAndView.

Alternatively, just call showForm() (or some such name)

I suggest your remove your implementation of formBackingObject and onSubmit as Spring will do that for you. I would also setSessionForm to true on the controller. By all means override onSubmit() and just stick a breakpoint there and see what the values are of the rrdBean in the session after a few requests.

Hope this helps.

Code:
<property name="sessionForm" value="true"/>
to your controller definition, the form object will be automatically bound to the session for you.
