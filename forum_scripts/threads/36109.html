First off all don't override setupForm just override the createForm method and let that method simply return your command object.

Code:
public class TransaccionEntradaAlmacenAction extends FormAction {

	private ConcreteCabeceraEntradaAlmacenImpl concreteCabeceraEntradaAlmacenImpl;
	
	public void setConcreteCabeceraEntradaAlmacenImpl(
			ConcreteCabeceraEntradaAlmacenImpl concreteCabeceraEntradaAlmacenImpl) {
		this.concreteCabeceraEntradaAlmacenImpl = concreteCabeceraEntradaAlmacenImpl;
	}

	@Override
	protected Object createFormObject(RequestContext context) throws Exception{
		return concreteCabeceraEntradaAlmacenImpl.getCabeceraEntradaAlmacenIdGeneradoBO();		
	}
}
That will bind your object to the correct scope and will use it as the formObject.

In your jsp you don't need the fmt tag to format the date, you already have a property editor. Also make sure that the setupForm method is part of your render-actions, this will re-register any property editors.

However the problem might also be solved by implementing the methods in the correct way as I suggested.
