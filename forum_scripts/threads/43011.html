Hi Saurav,

1. check whether the tag libs are avaible in the page that you are accesing.
- to do this just use some static html tags inside the <form:form> tag without any binding.

<form:form>
<h1>Testing</h1>
<p> test is ok</p>
</form:form>

If the tag lib is avaible then you can see the html without any errors.

2. If the tag libs are avaible then you should mention the command object name in your form before accessing its fields in the form tags. This can be done by setting the commandName attribute in form tag.

eg. <form:form commandName="yourCommandObjectName">
.................
............
</form:form>

alternatively you can use the full path when binding like this way

Name of the command object is assumed as 'yourCommandObjectName'.

<form:input path="yourCommandObjectName.heySaurav" />

Hi Saurav,

1 . Make sure that spring 2.0 jars in the /WEB-INF/lib directory. It is a must that not to mix other versions of spring jars with 2.0 jars.
2. This is how you can import the tag libs that are bundled with spring 2.0 jars.

Instead of putting
<%@ taglib uri="/tags/spring-form" prefix="form" %> line. If you use this then you have to add taglib entries in web.xml file.
add this line to top of your jsp,

<%@ taglib uri="http://www.springframework.org/tags/form" prefix="form" %>
Once that is done then check again whether the taglibs are avaible in ur jsp page.


3. You should return command object to your view.
.....handleRequestInternal(HttpServletRequest req){
....
...
return new ModelandView("viewName", "commandName", commandObject);

OR

3. Your command object should be configured like this in your spring configuration file. Here after I assume that you are using a controller that is extended from SimpleFormController in spring.

<bean id="yourController" class="YourClass">
<property name="formView" value="yourview1"/>
<property name="successView" value="yourview2"/>
<property name="commandClass" value="YourCommandClass"/>
<property name="commandName" value="commandName"/>
</bean>

In your controller class you should override 'formBackingObject()" method and return the command object.


4. Then your command object is available in your jsp and you can access that

<form:input path="commandName.filed"/>

Hi,
It seems that you are using the version 2.4. So you dont need to define your spring tag libs in web.xml file. Instead you can directly use that and only thing you need to do is mentioning the tag lib uri at the top of the page like this way.

<%@ taglib uri="http://www.springframework.org/tags/form" prefix="form" %>

But if you prefer the way that defining it in the web.xml (But the easy way is not defining it in the web.xml and use the tag lib by importing it at the top of the page as i mentioned) then make sure:

1. The spring form tag lib name is spring-form.tld and whether it exists in the
directory WEB-INF/tags/. So that should be configured in the web.xml file
as follows.
Code:
   <taglib>
     <taglib-uri>/tags/form</taglib-uri>
     <taglib-location>/WEB-INF/tags/spring-form.tld</taglib-location>
    </taglib-location>        
  </taglib>
Give a try and tell the status.

Hi,

Think that you are talking about the attribute "path". Actually its the path or the field name of the command object, that is binded to the view.

Say you want to bind a user object to the form. Also assume that User class has instace field of String named "username". You should write appropriate getters and setters for the filed you need to bind with the form.
In that case the path is "username".

You can give a name to your command object. If so, you need to mention that in the <form:form> tag as follows.

<form:form commandName="yourCommandObjectName">. Value of the "commandName" attribute is 'command' by default.
