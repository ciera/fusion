There are several things you can take a look at:
- Use a servlet error page (defined in web.xml).
- A Spring MVC HandlerExceptionResolver.
- SWF offers FlowExecutionExceptionHandler, which allows you to decide what to do when an exception occurs, either at the state level or the flow level. Check the "on-exception" attribute of the <transition> element and the <exception-handler> sub element of the <flow> element.

Another option you have is ofcourse have a Action superclass that deals with the exception similarly to the way you were doing in your Struts app. When the action catches the exception, it put's it in flash scope and signals an error event that will be intercepted by the flow, moving it back to the source state.

Quote:
the problem is that i cant see the error messages of a ficticius exception like a duplication for an id for example
You're constructing a BindException but you're not actually putting it anywhere. Try using the FormObjectAccessor helper class that comes with SWF. You can do stuff like this:

Code:
new FormObjectAccessor(context).putFormErrors(errors, ScopeType.FLASH);
Another thing you could do instead of creating a new Errors instance is to locate the Errors instance prepared by the FormAction and adding errors to that. The FormObjectAccessor can also help you in doing that.

Quote:
is correct the arguments for the BindException constructor ???, really what values should i write??
No. The first parameter should be the target object (normally the form object) and the second parameter should be the name of the target object. It is that name that you need to use in your spring:bind and spring:hasBindErrors tags.

Consult the Spring MVC documentation for more information on Spring MVC data binding and the spring: and form: taglibs. SWF just reuses the Spring MVC data binding machinery.
