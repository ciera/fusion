Yeah, the fact (not really problem) is that you are binding on a *collection*, not the individual elements in the collection.

I think you have two solutions:

in referenceData() create a map which contains all the possible elements, where the key is the element and the value is whether your backingObject.collection contains it or not, or use the JSTL fn:contains() function.

Example 1:
Code:
public Map referenceData(HttpServletRequest request, Object command, Errors errors) {
  Map contactsMap = new HashMap();
  List allContacts = getAllElements();
  Set selectedContactsSet = new HashSet(command.getSelectedContacts());
  for (Iterator i = allContacts.iterator(); i.hasNext(); ) {
    Contact contact = (Contact) i.next();
    contactsMap.put(contact, Boolean.valueOf(selectedContactsSet.contains(contact)));
  }
  Map model = new HashMap();
  model.put("allContacts", allContactsMap);
}
in jsp:

Code:
        <spring:bind path="alert.contacts">
            <select name="<c:out value="${status.expression}" />" MULTIPLE>
                <option value=""><fmt:message key="form.SelectOne"/></option>

                <c:forEach var="contactEntry" items="${allContacts}">
                    <option value="${contactEntry.key.id}" <c:if test="${contactEntry.value}">selected="selected"</c:if>>
                        <c:out value="${contactEntry.key.firstName}"/>&nbsp;<c:out value="${contactEntry.key.lastName}"/>
                    </option>
                </c:forEach>

            </select>
            <br/>
            <span class="error"><c:out value="${status.errorMessage}"/></span>
        </spring:bind>
Note: I just typed it in, so it *obviously* won't compile. Also, I *think* JSTL should be smart enough to realise that ${contentEntry.value} is a boolean, but if not then ${contentEntry.value == true}

And, by the way, I prefer the second solution Just wanted to exercise my fingers

So you have got the original "${}" problem sorted out?

Now, what is happening is the request parameters coming into Spring are *representations* of your object, i.e. the object ids. You need to convert these into the actual objects via a propertyEditor, which I presume you haven't done yet?

This is where it gets tricky You can either use a property editor to convert a single Id to your object, and this is what you would do if you were using an setCustomers(final Customer[] customers), but because you are using a Set, or a List (cannot remember) you will need to subclass CollectionsPropertyEditor: http://www.springframework.org/docs/...ionEditor.html.
