Ok I think I just got your question.

Before I get to it, you can initialize a bean as a flow variable that can hold those two params for you, and map onto it using your flow input mapper. Like this:

Code:
<flow>
    <var name="paramHolder" class="foo.ParamHolder"/>

    <input-mapper>
        <mapping source="param1" target="paramHolder.param1"/>
        <mapping source="param2" target="paramHolder.param2"/>
    </input-mapper>

     ..
</flow>
You can also use this simpler variant equivalent to your <mapping> code above:

Code:
<flow>
    <input-mapper>
        <input-attribute name="param1"/>
        <input-attribute name="param2"/>
    </input-mapper>
     ..
</flow>
Ok back to your question (the code you included confused me).

I assume you are using a bean-action. Yes, if one of the arguments is null it won't be able to resolve the method as the signature is unknown. To work around this, specify the 'type' attribute on your method argument definition like this:

Code:
<bean-action bean="foo" method="bar">
    <method-arguments>
        <argument expression="flowScope.param1" parameter-type="string"/>
        <argument expression="flowScope.param2" parameter-type="example.SomeOtherType"/>
    </method-arguments>
</bean-action>
