formBackingObject should be used when you typically want your command object to be prefetched with values before showing it (the first time) using your SimpleFormController.
Useful for EditXXX use cases.

referenceData is typically used to populate a list of values (LOVs) in your listboxes, or checkboxes. You cant have this in domain model as they usually are static data loaded from db.

If session variable is set to true, the command object is retrieved from session for subsequent calls (but first time formBackingObject) is still called.

A common alternative is to use a PRG approach, and on form submit, have the success view be a redirect back to the form view. This avoids the issue of a messy and possibly redundant onSubmit handler repopulating form data, avoids the "accidental reload/repost" issue, along with a number of other niceties.

I think I see the problem... The command that is posted needs to be put somewhere so that you can retrieve this after the redirect (session for example).

You need to change the following code.

override a different onSubmit() method

Code:
protected ModelAndView onSubmit(
            HttpServletRequest request,    HttpServletResponse response, Object command,    BindException errors)
            throws Exception {
request.getSession().setAttribute("myForm", command);
return new ModelAndView(new RedirectView(getSuccessView()));
}
override formBackingObject
Code:
protected Object formBackingObject(HttpServletRequest request) throws Exception {
        Object command = request.getSession().getAttribute("myForm");
        //remove from session...
        request.getSession().setAttribute("myForm", null);
        if (command == null) {
           command = createCommand();
        }
        return command;
    }
The problem is after the redirect formBackingObject is called and it creates a brand new form by default.

Hope this helps...

That is a solution. However you could also call the showForm method from your onSubmit method.

Code:
protected ModelAndView onSubmit(HttpServletRequest request, HttpServletResponse response, Object command, BindException errors) throws Exception {		
	return showForm(request, errors, getSuccesView());
}
This renders the the view with the new jsp without issueing a redirect.
