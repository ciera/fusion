There is no built-in function explicitly designed to do this. However, you have a few options:

- You could use a flow start Action or FlowExecutionListener to assert that the flow is running as a subflow.
- If the subflow is only used by a single parent flow you could make that subflow an inner subflow of that parent flow.

You can check to see if a flow (session) is running as a top-level flow started directly from the browser by doing context.getFlowExecutionContext().getActiveSession ().isRoot() where 'context' is a RequestContext object.

Stop the flow by throwing an exception. You can handle that exception in the web framework you're using, e.g. with a HandlerExceptionResolver in SpringMVC.
