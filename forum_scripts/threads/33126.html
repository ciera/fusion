You have a mistake in your c:if of in your spring:bind

Code:
<c:if test="${adj.approval_status!='R' && adj.approval_status!='A' }">
The above assumes you have a method named getApproval_status in your MultiAdjustment class.

Code:
<spring:bind path="command.multiAdjustments[${loopStatus.index}].approvalStatus">
This one assumes a getApprovalStatus method in your MultiAdjustment class.

So make sure that both methods are there or change the statements to include the correct getMethods.

You have a list of objects those objects need to have a get/set method.

So if you have a list of instances of DummyObject your DummyObject would have to implement a get/set method.

Code:
public class DummyObject
   
private String approvalStatus;

public void setApprovalStatus(String approvalStatus) {
    this.approvalStatus=approvalStatus;
}

public String getApprovalStatus() {
   return this.approvalStatus;
}
In your code you are binding to instance x of your list. If loopStatus.index has a value of 3 you are binding to the third element in the multiAdjustments property (a List in your case) of your command object.

That object (the on on the third position) needs to have a get/set method.

Code:
<spring:bind path="command.multiAdjustments[${loopStatus.index}].approvalStatus">
Just another question to clear up some things. What do you need/want to bind? do you want to put a value in approvalStatus of your command object, or do you want to put a value in a object contained in the multiAdjustments List.

FWIW the error in your original post has nothing to do with your code. Look in JIRA for NoSuchMethodError. Depending on what version of Spring you're using, and which container (I assume non-JSP 2.0-compliant, although it does appear you're using Weblogic). For example, bug SPR-2628 and SPR-2427 affect versions 1.2.x and 2.0 RC 3, respectively. This could be caused by something as simple as including jsp2x.jar in your web application's classpath (for e.g. for MyFaces support), when running on a non-JSP 2.0 container.

The reason for the error is that your PageContext is not a JSP 2.0 compliant one (is it? what version of WL are you running in? What version of the servlet spec do you declare in web.xml?), yet somehow you are picking up the jsp 2.0 class files in your web app's classpath. Make sure the JSP 2.0 class files *are not included in your web application at all* (as Juergen states in the bug, this is a violation of the relevant spec anyway) otherwise you will continue to run into this issue. I was not able to find any other workaround to the error - it's a bug in the way Spring detects what EL evaluator to use.

Well, considering that JSP 2.0 wasn't supported until J2EE 1.4 (servlet/webapp spec 2.4), you might try bumping the webapp version to 2.4:
Code:
<web-app xmlns="http://java.sun.com/xml/ns/j2ee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation=
           "http://java.sun.com/xml/ns/j2ee web-app_2_4.xsd"
         version="2.4">
