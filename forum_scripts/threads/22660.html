I would ask the question "what is valid data"? For example, you might decide that when creating a new (e.g.) User, a "valid" request is on where all the fields are not null. Then in onSubmit you try and create it, if it fails, you want to go back to the formView? I would suggest that actually "valid" data is one where all the fields are not null *and the username doesn't already exist*.

What I am suggesting is that your validator checks to make sure that the request is valid, but also that the operation will succeed. Does that make sense?

Quote:
On a partially-related question, how do Spring-y people normally redirect to another controller? I can pass in a URL and add it to the context path, but I'd rather pass in the id of a defined controller.
I would say it is best practice to always pass in a logical view name, that view name is then resolved to a RedirectView.

Controllers that do the redirecting are not such a good thing (IMHO)

What I ended up doing was checking the results of my service call and doing a RedirectView to a landing page inside the "full" onSubmit method override.


Quote:
I didn't see how to do that--if I'm using a SimpleUrlHandlerMapping is this even possible, or should I bail on that mapper and add another layer of abstraction such that I can have something called "homePage"? If so, which controller and how do I create a redirect out of it?
Sure, you simply define a view which is an instance of RedirectView and contains the url of the controller you want to redirect to. I always use ResourceBundleViewResolver, so you would do something like (assuming you are also using BeanNameUrlHandlerMapping:

Code:
  <bean name="yourUseCaseUrl.htm" class="yourSimpleFormController">
    ...
    <property name="successView" value="yourUseCaseUrlSuccessView"/>
  </bean>
 
  <bean name="urlYouWantToRedirectTo.htm" class="yourController"/>

and then in your view definition:

yourUseCaseUrlSuccessView.class=org.springframework....RedirectView
yourUseCaseUrlSuccessView.url=urlYouWantToRedirectTo.htm
I am not at my work machine so the above is from the top of my head but you get the idea.
